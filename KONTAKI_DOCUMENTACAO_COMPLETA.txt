1. VISÃO GERAL E FILOSOFIA DO PRODUTO
================================================================================

O Kontaki é uma aplicação web progressiva (PWA) de gestão de negócios projetada
especificamente para pequenas lojas, mercearias, supermercados e negócios de
varejo em contextos onde a conectividade à Internet é intermitente ou cara.

FILOSOFIA PRINCIPAL: Offline-First
- Os dados são armazenados primeiramente no dispositivo (IndexedDB)
- A sincronização com servidor é opcional e não é necessária para funcionamento
- O app funciona completamente offline sem perda de funcionalidades essenciais
- A sincronização é apenas para backup e segurança

PÚBLICO-ALVO:
- Proprietários de pequenas lojas
- Mercearias e vendedores de varejo
- Negócios com múltiplos funcionários
- Usuários com conexão de Internet instável

================================================================================

ARQUITETURA TÉCNICA:
├── Frontend (SPA - Single Page Application)
│   └── HTML5 / CSS3 / JavaScript Vanilla
│
├── Storage Local
│   └── IndexedDB (persistência offline)
│
├── Service Worker (sw.js)
│   └── Cache-first strategy para assets
│   └── Funciona offline completamente
│
├── Backend Opcional
│   └── Apenas para backup remoto
│   └── Sincronização encriptada
│

    └── Responsivo para todos os tamanhos
    └── Touch-friendly UI/UX


STACK TECNOLÓGICO:
- Linguagem: JavaScript (ES6+)
- Framework Front-end: Vanilla JS (sem frameworks pesados)
- Storage: IndexedDB + LocalStorage
- Service Worker: Offline-first caching
- Estilos: CSS3 com suporte a dark mode
- Ícones: Lucide Icons
- Gráficos: Chart.js (para dashboards)
- Impressão: jsPDF para relatórios em PDF
- QR Code: Suporte nativo para verificação de produtos


PADRÃO DE DESIGN: Flat Design Moderno
- Paleta de cores: Roxo (#5b21b6) como cor primária
- Interface limpa e intuitiva
- Modo claro/escuro nativo
- Responsividade total
- Navegação por tabs + submenu
- Feedback visual imediato

6. OFFLINE-FIRST: COMO FUNCIONA
================================================================================

6.1 ARMAZENAMENTO LOCAL
─────────────────────────────────────────────────────────────────────────────

INDEXEDDB:
- Base de dados local no navegador
- Capacidade: ~50MB por aplicação (pode pedir mais)
- Estrutura:
  ├── products (catálogo)
  ├── sales (vendas realizadas)
  ├── saleItems (itens de cada venda)
  ├── fiado (clientes em crédito)
  ├── stockMovements (histórico de movimento)
  ├── expenses (despesas)
  ├── users (utilizadores)
  ├── settings (configurações)
  ├── accountingArchive (períodos anteriores)
  └── logs (auditoria)

LOCALSTORAGE:
- Dados adicionais:
  ├── Carrinho temporário
  ├── Preferências de UI
  ├── Last sync timestamp
  └── Dark mode preference


6.2 FUNCIONAMENTO SEM INTERNET
─────────────────────────────────────────────────────────────────────────────

OPERAÇÕES COMPLETAMENTE OFFLINE:
✓ Realizar vendas
✓ Pesquisar produtos
✓ Consultar stock
✓ Ver histórico
✓ Registar despesas
✓ Gerir clientes fiados
✓ Imprimir recibos
✓ Gerar relatórios locais
✓ Ver dados contabilísticos

SERVICE WORKER:
- Cache-first strategy para assets
- Todos os arquivos em cache
- Sem requisições de rede necessárias
- Atualização automática quando online


6.3 SINCRONIZAÇÃO COM BACKUP
─────────────────────────────────────────────────────────────────────────────

INÍCIO:
- Manual: Clique em "Sincronizar"
- Automático: A cada 30 minutos (se online)
- Ao abrir/fechar turno

PROCESSO:
1. Detectar mudanças locais
2. Encriptar dados
3. Enviar para servidor
4. Receber confirmação
5. Atualizar timestamp de sync
6. Notificar sucesso

COLISÕES:
- Última escrita vence
- Versioning automático
- Sem perda de dados


6.4 RECUPERAÇÃO DE DADOS
─────────────────────────────────────────────────────────────────────────────

CENÁRIO: Perda local de dados

SOLUÇÃO:
1. App detecta IndexedDB vazio/corrompido
2. Oferece restauração de backup
3. Restauração completa de dados
4. Confirmação de integridade
5. Resumo de dados restaurados

PROTEÇÃO:
✓ Backup automático a cada sincronização
✓ Múltiplas versões mantidas
✓ Recuperação de até 30 dias
